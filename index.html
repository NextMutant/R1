Mandatory vs optional

● Must-have (non-negotiable)

TypeScript React Git / GitHub REST APIs PostgreSQL Docker GitHub Actions JWT Auth LangChain / LangGraph PyTorch HuggingFace System Design LeetCode Mediums AWS / GCP basics Tailwind CSS Redis

○ Nice-to-have (differentiators)

Kubernetes Terraform GraphQL Next.js MongoDB MLflow Multi-agent patterns Dynamic programming Microservices Serverless Kafka gRPC Cypress E2E Framer Motion Weights & Biases
DAILY STRUCTURE

First 2–2.5h on the week's primary topic. 1h on a side topic (e.g. LeetCode in Phase 1). Final hour reviewing, writing code, or documenting. Weekends for project work only — never introduce new concepts.

IF YOU GET BEHIND

TypeScript, React, REST APIs, Docker, and LangGraph appear in almost every JD. Deep learning and Kubernetes can slip to "working knowledge" level without costing you offers.

THE CAPSTONE RULE

Week 10's capstone — React + FastAPI + LangGraph, deployed, with CI/CD — answers 80% of portfolio questions across all three role types in a single demo. Keep it focused and polished, not feature-heavy.

SYSTEM DESIGN TIP

Start practising system design out loud from Week 8. Talking through designs — even to yourself or a rubber duck — is what converts reading into actual interview performance.
